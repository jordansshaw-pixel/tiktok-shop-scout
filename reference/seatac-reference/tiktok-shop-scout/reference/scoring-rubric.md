# Scoring Rubric

Score every product 1–5 on all six dimensions. Total: 6–30. Decision thresholds live in `rules.md` §3 — this file only defines what each score means.

| Dimension | 1 (Poor) | 3 (Acceptable) | 5 (Strong) |
|-----------|----------|----------------|------------|
| **Commission rate** | < 5% | 8–12% | > 15% |
| **Sales velocity** | Declining | Flat or rising slowly | Rising fast (week-over-week growth visible) |
| **Creator saturation** | > 2000 active affiliates | 500–2000 | < 500 |
| **Content angle** | No clear hook — "it's a nice product" | Generic demo possible | A specific scroll-stopping hook already exists or is obvious |
| **ICP fit** | Wrong audience entirely | Partial fit (right buyer, weak motive) | Direct match to a felt need in `audience-profile.md` |
| **Price point** | < $8 (AOV floor) or > $80 (hesitation zone) | $8–$15 or $60–$80 | $15–$60 (impulse sweet spot) |

## Scoring discipline

- **Even numbers are allowed** (2 and 4) when a product sits between anchors. Don't agonize — the anchors are the calibration, not a straitjacket.
- **Estimated values score one lower.** If you estimate commission at "probably 10–12%," score it 2, not 3, and mark `(est.)`. Confirmed data earns the full score.
- **Velocity needs a stated signal.** "Rising" requires evidence: climbing sales count on the listing, creator count growing, search/hashtag volume up. No signal = score it flat (3 max).
- **Content angle is scored against the top existing videos**, not in a vacuum. A great hook that 50 creators already use is a 3, not a 5.
- **ICP fit cites the profile.** A 4–5 must name which felt need or buying trigger from `audience-profile.md` the product hits. If you can't name one, it isn't a 4.

## Worked micro-example

Mini blender, $24, 12% commission (confirmed), ~350 creators, sales climbing, hook: "I stopped buying $9 smoothies."

Commission 3 · Velocity 4 · Saturation 4 · Angle 4 · ICP fit 4 (kitchen time-saver, "it's only $24" price trigger) · Price 5 → **24/30, commission confirmed → PROMOTE.**
